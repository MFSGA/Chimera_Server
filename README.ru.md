## Описание проекта

### Цель проекта
Chimera_Server — это сетевое ядро на Rust, цель которого заключается в максимальной совместимости с
проектом **xray-core** с открытым исходным кодом. На текущем этапе основной акцент сделан на
**конфигурации inbound и поведении входящих протоколов**, чтобы максимально повторять имена полей,
значения по умолчанию, процесс рукопожатия и семантику выполнения в upstream xray-core.

### Текущее состояние
- Логика разбора входящих конфигураций и диспетчеризации активно развивается; в приоритете VMess,
  VLESS, Trojan и другие распространенные входящие протоколы.
- Модули outbound, routing и policy все еще находятся в стадии проектирования; пока доступны только
  базовые или заглушечные реализации.
- Репозиторий разделен на `chimera_server_app` (точка входа приложения), `chimera_server_lib`
  (основная библиотека) и `chimera_cli` (вспомогательная CLI-утилита).

### Референсная реализация
Чтобы сократить разрыв с xray-core, часть логики разбора inbound и обработки протоколов опирается на
сообщественный проект [shoes](https://github.com/cfal/shoes). Его подходы к рукопожатию, поведению по
умолчанию и обработке ошибок полезны как референс для этого проекта.

### Быстрый старт
1. Установите последнюю стабильную версию Rust через `rustup`.
2. Клонируйте репозиторий и перейдите в каталог проекта.
3. Запустите сервис командой:

```bash
cargo run --package chimera_server_app -- --config path/to/config.json5
```

4. Для горячей перезагрузки используйте `start.sh` или `start_server.ps1`, предварительно проверив их
   предположения об окружении.

### CLI-утилита
`chimera_cli` включает вспомогательный бинарник `chimera-cli`, который повторяет сценарий
использования `xray x25519`. Из корня workspace выполните:

```bash
cargo run -p chimera_cli -- x25519 --count 1 --format base64
```

Команда выводит пару приватный/публичный ключ в том же порядке, что и xray-core, и поддерживает
форматы `base64` и `hex`.

### Конфигурация
- Файлы конфигурации следуют структуре xray-core; сейчас основной упор сделан на массив `inbounds`
  и связанные настройки протоколов и транспорта.
- Парсеры стараются повторять имена полей и значения по умолчанию upstream, чтобы существующие
  конфигурации xray-core можно было адаптировать с минимальными изменениями.
- В настоящее время предпочтителен формат `json5`, а каталог `examples/` содержит несколько
  готовых примеров.

### Пример inbound
```json
{
  "inbounds": [
    {
      "tag": "vmess-tcp",
      "protocol": "vmess",
      "listen": "0.0.0.0",
      "port": 10086,
      "settings": {
        "clients": [
          {
            "id": "YOUR-UUID",
            "alterId": 0
          }
        ]
      },
      "streamSettings": {
        "network": "tcp"
      }
    }
  ]
}
```
